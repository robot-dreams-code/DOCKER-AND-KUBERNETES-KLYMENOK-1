## Rollout
Після зміни параметра у конфігмапі рестартую деплоймент, щоб примінити зміни:
```sh
kubectl rollout restart deployment course-app -n dev
deployment.apps/course-app restarted
```

## RollingUpdate Deployment

Схоже на того, що під куполом цирку намагається не впасти.  
Бо якщо **maxUnavailable: 0 maxSurge: 1** - впираємося у ресурси кластера, створюючи додаткові поди і навантаження. Але без даунтайма.
Якщо **maxUnavailable: 1 maxSurge: 0** - впираємося у можливості контейнера, може не прожувати трафік і впасти. Але кластер точно не впаде.
Якщо **maxUnavailable: 25% maxSurge: 25%** - оновлення буде швидким, але болісним)) І кластер роздується, і аплікуха кряхтітиме
